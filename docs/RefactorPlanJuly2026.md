# Refactoring Plan: AdamsQuotes

This is a refactoring plan generated by GLM 5.2 to cleanup this very old project.

## Current State

Four standalone scripts forming a pipeline:

| Script | Stage | Purpose |
|---|---|---|
| `addTagsToRawQuotes.py` | 1 | Raw quotes → semi-processed tagged markdown |
| `processNewQuotes.py` | 1 (alt) | New-format raw quotes → tagged markdown |
| `processTaggedQuotesWithLLM.py` | 1.5 | LLM cleanup of tagged markdown |
| `processQuotes.py` | 2 | Tagged markdown → styled HTML page |

## Key Issues Identified

1. **camelCase everywhere** — `idForQuote`, `generateTitle`, `QuotesList`, `writeQuoteAttributeToFile`, `linesFromFile`, `QuoteTitles`, `addClassToTD`, `writeTableOfContents`, `processQuotes`, `printQuotes`, `generateHtml`, `writeQuotes`
2. **`Quote` class with manual `__init__`** — should be a `dataclass`
3. **`ProcessQuotes` class** — holds state but its methods are mostly pure transforms; should be module-level functions
4. **pandas dependency** — used only for a two-column HTML table in `writeTableOfContents`
5. **CLI mixed with business logic** — `argparse` + `if __name__ == "__main__"` live in the same files as the logic
6. **Module-level side effect** — `processQuotes.py` has `parser = _build_parser()` at module level, which runs on import (tests even rely on it: `from processQuotes import parser`)
7. **Duplicate tagged-output logic** — `_write_quote` (addTagsToRawQuotes) vs `_build_quote_output` (processNewQuotes) produce the same format with slightly different cleaning
8. **Duplicate URL detection** — `_is_url`/`_contains_url` (processNewQuotes) vs inline `"http" in line` (addTagsToRawQuotes)
9. **Duplicate text cleaning** — `_clean_text`/`_unwrap_paragraphs` (processNewQuotes) vs nothing in addTagsToRawQuotes
10. **Tests import from top-level modules** — `from processQuotes import Quote`
11. **`writeQuotes` uses relative path** — `open("Header.html")` breaks if cwd differs
12. **`generateTitle` mutates `self` and returns `self`** — fluent but unusual pattern

## Phases

### Phase 0 — Preparation

1. Create a feature branch: `git checkout -b refactor/pythonic-structure`
2. Verify tests pass on current code: `uv run pytest` (green baseline)
3. Commit current state as a reference point

### Phase 1 — Extract shared utilities (low risk, high reuse)

The four scripts duplicate a lot of logic. Start by extracting the common pieces into shared modules.

**Create `adamsquotes/text_utils.py`** — consolidate:
- `_is_url()` / `_contains_url()` from `processNewQuotes.py` (currently duplicated as inline `"http" in line` in `addTagsToRawQuotes.py`)
- `_clean_text()` and `_unwrap_paragraphs()` from `processNewQuotes.py`
- `_is_dash_line()`, `_is_italic_marker()`, `_is_title_like()`, `_ends_with_sentence_punct()` from `processNewQuotes.py`

**Create `adamsquotes/types.py`** — shared constants:
- `KNOWN_AUTHORS` list (currently in `addTagsToRawQuotes.py`)
- Tag schema constants (`QUOTE_TAG = "*quote:*"`, etc.) — currently hardcoded as string literals in 3 files
- `DEFAULT_INPUT` / `DEFAULT_OUTPUT` paths

**Create `adamsquotes/models.py`** — the `Quote` dataclass:

```python
from dataclasses import dataclass, field
from random import randint


@dataclass
class Quote:
    quote: str = ""
    source: str = ""
    note: str = ""
    link: str = ""
    author: str = ""
    id_for_quote: int = field(default_factory=lambda: randint(10000, 99999))
    title: str = ""

    def generate_title(self) -> None:  # mutates self, no return
        ...
```

### Phase 3 — Decompose `ProcessQuotes` into functions

The `ProcessQuotes` class holds state but its methods are mostly pure transforms. Replace it with module-level functions in `adamsquotes/pipeline/html_renderer.py`:

```python
def parse_quotes(text: str) -> list[Quote]: ...
def render_quote_html(quote: Quote) -> str: ...
def render_toc(quotes: list[Quote]) -> str: ...
def generate_html(quotes: list[Quote], header_template: str) -> str: ...
def write_html(quotes: list[Quote], output_path: Path, header_path: Path) -> None: ...
```

**Fix the `Header.html` path bug**: `writeQuotes()` currently does `open("Header.html")` with a relative path. Pass `header_path` as a parameter instead.

---

### Phase 4 — Remove the pandas dependency

`writeTableOfContents()` uses `pd.DataFrame(...).to_html()` for a two-column table. Replace with a simple string join:

```python
def render_toc(quotes: list[Quote]) -> str:
    rows = "\n".join(
        f'  <tr><td class="column0"><a class="tableOfContents" href="#{q.id_for_quote}">{q.title}</a></td>'
        f'<td class="column1">{q.source[:30]}{"..." if len(q.source) > 30 else ""}</td></tr>'
        for q in quotes
    )
    return (
        '<div class="table-of-contents">\n'
        "<table>\n"
        '<tr><th class="column0">title</th><th class="column1">source</th></tr>\n'
        f"{rows}\n"
        "</table>\n"
        "</div>\n"
    )
```

Then remove `pandas` from `pyproject.toml` dependencies.

---

### Phase 5 — Separate CLI from logic

Each script currently mixes `argparse` + `if __name__ == "__main__"` with business logic. Move CLIs to `adamsquotes/cli/`:

```
adamsquotes/cli/
  __init__.py
  tagger.py        # argparse → calls pipeline.tagger.quote_handler()
  converter.py     # argparse → calls pipeline.converter.process_file()
  llm_cleaner.py   # argparse → calls pipeline.llm_cleaner.main()
  renderer.py      # argparse → calls pipeline.html_renderer.write_html()
```

**Fix the module-level side effect**: `processQuotes.py` has `parser = _build_parser()` at module level, which runs on import (the tests even rely on it: `from processQuotes import parser`). Move parser construction inside `if __name__ == "__main__":` or a `main()` function.

Run via: `uv run python -m adamsquotes.cli.renderer --quotes_input ...`

---

### Phase 6 — Consolidate duplicate tagged-output logic

`addTagsToRawQuotes._write_quote()` and `processNewQuotes._build_quote_output()` produce the same tagged format but with slightly different cleaning. Unify into one function in `adamsquotes/models.py` or `adamsquotes/text_utils.py`:

```python
def format_tagged_quote(
    quote: str,
    source: str,
    author: str,
    link: str,
    note: str,
    clean: bool = True,
) -> str:
    if clean:
        quote, source, author, link, note = map(
            _clean_text, (quote, source, author, link, note)
        )
        quote = _unwrap_paragraphs(quote)
    return (
        f"*quote:*\t{quote}\n"
        f"*source:*\t{source}\n"
        f"*author:*\t{author}\n"
        f"*link:*\t{link}\n"
        f"*note:*\t{note}\n"
    )
```

## Suggested Execution Order

| Step | Risk | Why this order |
|---|---|---|
| 1. Extract `text_utils.py` + `types.py` | Low | Pure additions, no behavior change |
| 2. Extract `models.py` (Quote dataclass) | Low | Drop-in replacement |
| 3. Rename to snake_case | Medium | Mechanical but touches tests |
| 4. Decompose `ProcessQuotes` → functions | Medium | Changes API, tests must follow |
| 5. Remove pandas | Low | Isolated to `render_toc` |
| 6. Separate CLI | Low | Just moving code |
| 7. Consolidate duplicate logic | Medium | Merges two code paths |
| 8. Reorganize tests | Low | Follows code changes |
| 9. Update pyproject + README | Low | Documentation |
| 10. Delete old scripts | Low | Cleanup |

**Run `uv run pytest` after each phase** to catch regressions early.
